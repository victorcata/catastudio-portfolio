USABILITY ANALYSIS AND INTERACTION DESIGN

I believe that as a developer I not only have to improve my technical skills, I also believe that we work for people and we must to understand our users, how they work, what they need and how we can help them develop software.

Since 2014 I've been thinking more about usability and interaction design, or how people interact with machines. So when I'm designing and developing software I'm thinking about how I can reduce friction between the UI and the user.

IMPROVING UI COMPONENTS

Since I've been working in the Avanade UX department I've been able to put into practice what I've been studying in Interaction Design and I've been learning from my UX colleagues. I think my mind as a developer and my experience working on site on a big client with hundreds of users helped me to understand how a person interact with the software.

I like to think about UI components and how I can improve them to solve usability problems. These screenshots are from different controls that I've been trying to improve to apply later in our UI designs. These components include list inlays in which the user has to view more than one section at the same time, hierarchical multi selects, wizards where the user must know how many data left to give to finish the process, archive list with a preview of the filename or an example of calendar with appointments and zoom, among others.
